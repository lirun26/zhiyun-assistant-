## 🎯 二、游戏玩法

### 2.1 核心玩法

```
成语填空 → 下一关 → 获得金币/体力
     ↓
  错误 → 消耗体力/看广告复活
```

### 2.2 玩法类型

| 玩法 | 描述 | 难度 |
|------|------|------|
| **成语填空** | 给出一个成语，缺少1-2个字 | ⭐ |
| **看图猜成语** | 显示图片/emoji，猜成语 | ⭐⭐ |
| **成语接龙** | 给出成语的最后一个字，接下一个 | ⭐⭐⭐ |
| **连线成句** | 把乱序的字连成成语 | ⭐⭐⭐⭐ |
| **成语纠错** | 找出错别字并纠正 | ⭐⭐ |

### 2.3 体力系统

- 初始体力：10点
- 每关消耗：1点
- 恢复方式：时间恢复（1点/5分钟）、看广告、购买
- 体力上限：20点

### 2.4 闯关系统

- 总关卡：1000+关
- 难度递进：简单→中等→困难→专家
- 主题分类：动物、人物、自然、典故

## 🏗️ 三、系统架构

### 3.1 页面结构

```
首页
├── 开始游戏 → 关卡选择 → 答题页 → 结果页
├── 每日挑战 → 定时刷新题目
├── 成语接龙 → 好友对战 / AI对战
├── 收藏夹 → 复习已学成语
└── 排行榜 → 好友排行 / 全国排行
```

### 3.2 数据结构

```javascript
// 用户数据
User {
  id: string,
  name: string,
  level: number,        // 当前关卡
  coin: number,         // 金币
  energy: number,       // 体力
  favorites: string[],  // 收藏的成语
  stats: {
    correct: number,
    wrong: number,
    streak: number      // 连续正确
  }
}

// 成语题库
Question {
  id: string,
  type: 'fill' | 'guess' | 'idiom' | 'line' | 'correct',
  question: string,     // 题干
  answer: string,      // 答案
  hints: string[],      // 提示（首字、含义等）
  difficulty: 1-5,
  category: string     // 分类
}
```
