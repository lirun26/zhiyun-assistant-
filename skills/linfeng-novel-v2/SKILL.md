# 📋 小说创作规范 — 番茄签约核心 v2.0

**版本**: v2.0 (2026-04-15)
**整合**: linfeng-novel + novel-ai-prompts（Hermes版）
**目标**: 签约成功，不是完美主义

---

## 一、番茄小说平台签约核心要求（最低门槛）

1. **字数**：每章 **2500-3500 字**（低于2000字直接拒，上限3500禁止超）
2. **节奏**：前三章必须出现第一个冲突/爽点/钩子
3. **开篇**：第1句就要有事件/冲突，不能从环境描写开始
4. **有CP**：无CP难度极高，建议1V1感情线
5. **逻辑自洽**：资金、物品、人物关系前后一致
6. **AI味检测**：有明显AI特征会被拒稿

---

## 二、番茄拒稿原因归纳

### ❌ 致命问题（P0 — 直接拒稿）

#### 1. 字数严重不达标
| 章节 | 实际字数 | 番茄要求 | 差距 |
|------|---------|---------|------|
| 第9章 | ~1459字 | 2000字+ | ❌ 不足40% |
| 第10章 | ~1150字 | 2000字+ | ❌ 不足60% |

**后果**：连续5章字数不达标，编辑直接判定"章节过短，水分过多，拒稿"

#### 2. 开篇前三章无有效冲突
番茄爆款开篇公式：
```
第1句：事件冲突（不是氛围）
第3段：主角目标亮相
第1页：第一个障碍/危机出现
```

#### 3. AI写作特征太明显（番茄有AI检测）
- 大量内心独白替代动作描写
- "绝非...绝不...绝不再..."高频否定排比
- 固定句式："嘴角勾起一抹..."反复出现
- 每章结尾有"本章总结"和"下章预告"
- "末世爆发前XX天"倒计时句式反复出现

#### 4. 平台定位偏差
- 末世文属于**冷门赛道**，番茄主流是**都市言情/甜宠/1V1**
- 建议：要么改平台，要么加感情线

---

### ❌ 逻辑硬伤（P0）

#### 5. 资金逻辑矛盾
- 现金15.5万，支出43万，缺口27.5万

#### 6. 物品来源不明
- 电棍、匕首等武器没有出现就使用了
- 每件物品必须"先出现，再使用"

#### 7. 交通方式前后矛盾
- 第9章骑摩托车，第12章写成"开车"

## 三、黄金开篇公式（前三章必须做到）

### 第1章
```
第1句：事件驱动（不是氛围描写）
前3段：主角身份+处境明朗
前1页：第一个冲突/危机出现
前3页：金手指/独特能力亮相
```

### 第2章
```
主角用能力解决第一个小问题（验证金手指）
埋1-2个后续伏笔
```

### 第3章
```
出现第一个反派/障碍
建立本章核心冲突
```

## 四、AI味识别清单（每章发布前自检）

| 禁止项 | 说明 |
|:---|:---|
| "本章总结/下章预告" | ❌ AI标志 |
| "绝非...绝不...绝不再..." | ❌ 排比句式 |
| 连续3段以上内心独白 | ❌ 应用行动展示 |
| "嘴角勾起一抹..." | ❌ 固定句式不超2次/章 |
| "突然/就在这时/猛然" | ❌ 转折词，用"下一秒"/"谁知"/"但"替换 |
| "他心中想到/不由得/不禁" | ❌ 弱势表达，用动作/对话展示 |

## 五、写作核心技巧

### 🎯 本质
> 好小说 = 让读者**想知道接下来发生什么**，不是文笔好，是钩子强

### ⚡ 开篇法则
| 法则 | 说明 |
|:---|:---|
| 第一句话就要出事 | 不能从平静日常开场 |
| 爽点要前置 | 前3章必须有1-2个让读者"卧槽"的点 |
| 重生反差 | 前世惨状+今生弥补，不写反差=浪费设定 |

### ✍️ 写作技巧
| 技巧 | 操作 |
|:---|:---|
| 对话 | 每句有潜台词，没有就删 |
| 打斗 | 招式名+反应+结果，3要素缺一不可 |
| 动作 | "他说"之前先写他做了什么 |
| 少用"的" | "他的手"→"手一抖"，主动句更有力 |
| 长句断行 | 2-3个逗号就断 |
| 呼吸节奏 | 紧3松1，全程紧绷会累 |

### 🔗 伏笔与节奏
| 原则 | 操作 |
|:---|:---|
| 伏笔原则 | 埋1-2条，收完再埋新的 |
| 反派不能弱 | 反派越强，赢的时候越爽 |
| CP写法 | 前20章并肩作战，20-30章才暧昧 |

## 六、AI辅助创作方法

### 1. 结构化提示词（不要单次对话）

❌ 差：
```
"帮我写一个玄幻小说第一章"
```

✅ 好（分步骤给任务）：
```
Step1: 生成大纲 → 人工筛选 → 修改
Step2: 按大纲展开第1章，要有开篇冲突
Step3: 检查：节奏是否够快？删除弱词？
```

### 2. 让AI扮演「读者」而不是「作者」

```
"你是一个资深小说编辑，专门负责签约审核。
阅读以下开头3000字，从以下维度评分：

1. 开篇钩子（0-25分）
2. 主角人设是否清晰（0-25分）
3. 前3章节奏（0-25分）
4. 商业潜力（0-25分）
5. 具体修改建议"
```

### 3. 分段生成策略（不要一次生成整章！）

```
第1段（500字）：开篇冲突
第2段（800字）：主角反应+金手指展示
第3段（800字）：第一个小高潮
第4段（500字）：结尾悬念
第5段（400字）：本章结尾
```

### 4. 提纲 + 大纲模式

| 阶段 | 内容 | AI/人工 |
|:---:|:---|:---:|
| 阶段一 | 生成完整提纲 | AI → 人工筛选 → 修改 |
| 阶段二 | 生成细纲 | 每章200字细纲 → 人工确认 |
| 阶段三 | 正文生成 | 分段生成 → 人工润色 |

## 七、快速检查命令

```bash
# 字数检查（2500-3500字）
wc -m 第XX章.md

# AI味检测
grep -c "总结\|预告\|绝非" 第XX章.md || echo "无AI味 ✅"
grep -c "突然\|就在这时\|猛然" 第XX章.md
grep -c "嘴角勾起一抹" 第XX章.md

# 固定句式检测
echo "=== 弱势表达检测 ===" 
grep -c "他心中\|不由得\|不禁" 第XX章.md

# 开篇检查
head -10 第XX章.md

# 结尾检查
tail -5 第XX章.md
```

## 十一、33维度审计（每章必检）

### A. 角色维度
| ID | 维度 | 检查 |
|:---:|:---|:---|
| A1 | 角色记忆 | 不能"记起"从未亲眼见过的事 |
| A2 | 角色状态 | 位置、情绪、体力必须一致 |
| A3 | 角色能力 | 技能使用不能超出当前境界 |
| A4 | 角色关系 | 对话态度符合关系亲疏 |
| A5 | 角色成长 | 性格变化必须有铺垫 |

### B. 连续性维度
| ID | 维度 | 检查 |
|:---:|:---|:---|
| B1 | 时间线 | 日期/时间顺序正确 |
| B2 | 地点 | 角色位置连贯 |
| B3 | 物品 | 获取/消耗有记录 |
| B4 | 环境 | 不能前后矛盾 |
| B5 | 势力分布 | 各势力位置不冲突 |
| B6 | 资源数量 | 金钱/物资不超出已有 |

### C. 伏笔维度
| ID | 维度 | 检查 |
|:---:|:---|:---|
| C1 | 伏笔埋设 | 每章最多2个新伏笔 |
| C2 | 伏笔回收 | 必须引用原始场景 |
| C3 | 伏笔逻辑 | 回收必须有因果关系 |
| C4 | 钩子设置 | 章节结尾必须有钩子 |

### D. 叙事维度
| ID | 维度 | 检查 |
|:---:|:---|:---|
| D1 | 节奏 | 高潮→缓冲→发展循环 |
| D2 | 信息密度 | 不能流水账 |
| D3 | 冲突设计 | 每章必须有核心冲突 |
| D4 | 悬念 | 关键信息延迟释放 |
| D5 | 视角 | 不能突然切换视角 |

### E. 语言维度
| ID | 维度 | 检查 |
|:---:|:---|:---|
| E1 | AI词汇 | 无"赋能/维度/赛道"等 |
| E2 | 句式单调 | "突然/就在这时"出现过多 |
| E3 | 感悟式结尾 | 禁止总结式结尾 |
| E4 | 感叹式结尾 | 禁止感叹号结尾 |
| E5 | 重复表达 | 同一描写不超2次 |

### F. 逻辑维度
| ID | 维度 | 检查 |
|:---:|:---|:---|
| F1 | 因果逻辑 | 事件必须有因果 |
| F2 | 战力逻辑 | 越级挑战必须有原因 |
| F3 | 情报逻辑 | 获取情报必须有来源 |
| F4 | 动机逻辑 | 角色行为必须有动机 |

### G. 类型维度
| ID | 维度 | 检查 |
|:---:|:---|:---|
| G1 | 修仙术语 | 都市类禁止修仙用语 |
| G2 | 都市术语 | 修仙类禁止现代用语 |
| G3 | 设定一致 | 境界体系不能混乱 |
| G4 | 地理真实 | 地名/距离必须合理 |
