# 📝 AI小说创作提示词模板库

> **版本**: v1.0 (2026-04-14)
> **来源**: 平台签约逻辑 + AI特性总结
> **用途**: 提升番茄等平台签约通过率

---

## 一、平台签约的本质要求

### 核心认知
签约编辑看的是**商业价值潜力**，不只是文笔：
- 前3万字要有强烈的"钩子"（冲突/悬念/爽点）
- 开头3000字决定80%的签约命运
- 题材是否有市场，受众是否够大
- 更新能力（每天4000-6000字）

### 黄金开篇公式
```
第1句：事件/冲突（不是氛围！）
前3段：主角身份+处境明朗
前1页(500字)：第一个冲突/危机出现
前3页(1500字)：金手指/独特能力亮相
```

## 二、AI创作质量提升方法

### 1. 结构化提示词（不要单次对话）

❌ 差：
```
"帮我写一个玄幻小说第一章"
```

✅ 好（分步骤给任务）：
```
Step1: "根据【题材：都市修仙】【核心爽点：逆袭打脸】
【主角人设：废物赘婿觉醒】【前10章主线：退婚流】生成小说大纲"

Step2: "按大纲展开第1章，要有【开篇冲突】
【3000字内出现第一个爽点】【用'然而下一秒'式转折】"

Step3: "检查本章：是否有足够冲突？
节奏是否够快？删除所有'然而'、'突然'、'就在这时'等弱词"
```

### 2. 让AI扮演「读者」而不是「作者」

提示词模板：
```
"你是一个资深小说编辑，专门负责[题材]签约审核。
阅读以下开头3000字，从以下维度评分并指出具体问题：

1. 开篇钩子（0-25分）
2. 主角人设是否清晰（0-25分）
3. 前3章节奏（0-25分）
4. 商业潜力/题材热度（0-25分）
5. 具体修改建议（每条都要有原文定位）"
```

### 3. 控制AI输出的技巧

- **限制字数**：每段800-1500字，方便平台编辑快速扫读
- **强制具体化**：
  - "禁止写心理描写"
  - "禁止写环境描写"
  - "每段必须有对话或动作"
- **分段生成再拼接**：不要一次生成整章，AI会越写越散

### 4. 提纲 + 大纲模式（最重要）

| 阶段 | 内容 | 要求 |
|:---:|:---|:---|
| 阶段一 | 生成完整提纲 | AI生成 → 人工筛选 → 修改 |
| 阶段二 | 生成细纲 | 每章200字细纲 → 人工确认节奏 |
| 阶段三 | 正文生成 | 按细纲分批生成 → 人工润色 |

## 三、具体提示词模板

### 开头黄金三章模板
```
"写一个[题材]小说开头三章，要求：
1. 第一章第500字内必须出现重大冲突或打脸
2. 每章结尾必须留悬念（用'就在这时'之外的写法）
3. 主角性格在前3000字内通过行为展示，不要直接描述
4. 删除所有'他心中想到''不由得''不禁'等弱势表达
5. 禁止在开头写超过2句的环境/风景描写
6. 每章至少要有3处具体冲突（语言冲突或行为冲突）"
```

### AI味检测提示词
```
"你是一个资深编辑，检查以下章节的AI写作特征：

[列出需检测的内容]

检测项：
1. 是否有"突然/就在这时/猛然"等突兀转折词（>3次为问题）
2. 是否有连续3段以上内心独白
3. 是否有固定句式（如"嘴角勾起一抹"）出现>2次
4. 是否有"绝非...绝不...绝不再..."排比句式
5. 是否有章节结尾的"总结"和"预告"
6. 是否有弱势表达："他心中想到"/"不由得"/"不禁"

对每项问题，引用原文具体句子说明"
```

### 签约评分提示词
```
"你是一个番茄小说签约审核编辑，评估以下开头是否值得签约：

[粘贴开头3000字]

评分维度（每项25分）：
1. 开篇钩子：是否能第一时间吸引读者？
2. 主角人设：是否清晰、有记忆点、让人想追？
3. 前3章节奏：冲突是否够密、爽点是否够早？
4. 商业潜力：题材是否热门、受众是否够大？
5. 更新能力：作者是否能做到日更4000+？

给出：
- 每项得分（0-25）
- 总分（0-100）
- 签约建议（建议签约/待观察/不建议）
- 具体修改建议（每条要有原文定位）"
```

## 四、人设强化提示词

### 标签化细节模板
```
"为以下主角写10个'标签化细节'（读者一看到这个细节就记住人设），
格式：【细节+人设含义】

示例：
- 主角永远把筷子横放在碗上 → 暗示家道中落的执念
- 说话从不看对方眼睛 → 暗示自卑但内心骄傲

要求：
- 细节要具体可感知
- 不要抽象性格描述
- 每个细节都要有画面感"
```

### 标签化细节示例

| 细节 | 人设含义 |
|:---|:---|
| 把筷子横放在碗上 | 家道中落的执念 |
| 说话从不看对方眼睛 | 自卑但内心骄傲 |
| 口袋里永远有糖 | 讨好型人格的补偿机制 |
| 遇到冲突先退一步 | 被欺负过的应激反应 |

## 六、实战流程建议

### 核心原则
**AI最擅长的是快速迭代，不是一步到位。**
用AI生成10个版本，人工选最好的再改，比让AI一次写好要靠谱得多。

### 标准流程

| 步骤 | 内容 | AI/人工 | 比重 |
|:---:|:---|:---:|:---:|
| 1 | 生成5-10个题材方向（带市场分析） | AI | 100% |
| 2 | 选定1个后，AI给出30章大纲 | AI | 80% |
| 3 | 人工审核大纲（**这步必须自己做**） | 人工 | 70% |
| 4 | AI按章生成正文 + 人工润色 | AI70%+人工30% | - |
| 5 | 前3章重点打磨，单独用编辑提示词审查 | 人工 | 100% |
| 6 | 发平台内投（不要直接发书，先内投过审） | - | - |

### 关键提醒
- 步骤3（大纲审核）决定**70%成败**，必须人工做
- 步骤5（前3章打磨）决定**80%签约命运**，重点投入
- 不要一次生成整章，分段生成+拼接

## 八、禁止项清单（每章必检）

| 禁止项 | 原因 | 替换建议 |
|:---|:---|:---|
| "突然/就在这时/猛然" | AI转折词，太突兀 | "下一秒"/"谁知"/"但" |
| "他心中想到/不由得/不禁" | 弱势表达 | 用动作/对话展示 |
| 连续3段以上内心独白 | 水文，节奏慢 | 用行动展示心理 |
| "嘴角勾起一抹..." | 固定句式 | "他笑了"/"他眼神一冷" |
| 章节末尾总结/预告 | AI标志 | 直接结尾 |
| 环境描写>2句/开头 | 水文 | 用情节代替环境 |

## 五、分段生成策略

### 不要一次生成整章！
```
❌ 错误：一次生成3000字（越写越散）

✅ 正确：分段生成
第1段（500字）：开篇冲突
第2段（800字）：主角反应+金手指展示
第3段（800字）：第一个小高潮
第4段（500字）：结尾悬念
第5段（400字）：本章结尾
```

### 每段生成后的检查
1. 是否有具体冲突？
2. 是否有对话或动作？
3. 是否删除了弱势表达？
4. 是否保持了节奏？

## 六、快速检查命令

```bash
# AI味检测
echo "===弱势表达检测===" && grep -c "突然\|就在这时\|猛然" 第XX章.md
echo "===内心独白检测===" && grep -c "他心中\|不由得\|不禁" 第XX章.md
echo "===固定句式检测===" && grep -c "嘴角勾起一抹" 第XX章.md

# 字数检查
wc -m 第XX章.md  # 确认2500-3500字

# 开篇检查
head -20 第XX章.md  # 确认第1句是事件/冲突
```
