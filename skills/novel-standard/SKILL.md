# novel-standard

小说创作标准化技能包 — 番茄/七猫/起点签约必备

## 核心功能

| 功能 | 说明 |
|-----|------|
| 平台标准 | 番茄/七猫/起点签约要求一览 |
| 字数控制 | 2500-3500字/章强制检查 |
| AI味检测 | 6项P0级别AI写作特征检测 |
| 逻辑检查 | 资金/物品/人物一致性 |
| 发布审批 | 章节发布前必须通过所有检查 |

## 工作流程

```
写作完成 → 字数检查 → AI味检测 → 逻辑检查 → 发布
    ↓          ↓           ↓           ↓
  补字数    不合格→修改   不合格→修改   通过→发布
```

## 字数标准（番茄平台）

| 类型 | 要求 |
|-----|------|
| 每章字数 | 2500-3500字 |
| 低于2000字 | 禁止发布 |
| 超过3500字 | 必须拆分 |

**快速判断**：WPS/Word字数统计，中文单字符，不计标点

## AI味识别清单（P0级 — 有任何一项直接修改）

| 检查项 | 模式 | 修复方式 |
|-------|------|---------|
| 本章总结/预告 | 包含"本章总结"或"下章预告" | 删除 |
| 排比否定句 | "绝非...绝不...绝不再..." | 改写为行动 |
| 固定句式 | "XX的嘴角勾起一抹..."出现>2次 | 换表达方式 |
| 倒计时句式 | "末世爆发前XX天"反复出现 | 改用事件推进 |
| 纯内心独白 | 连续3段以上内心独白 | 外化为动作/对话 |
| 上帝视角 | "全书第X章..."、"所有人没想到..." | 删除 |

## 逻辑检查清单（发布前必须验证）

| 检查项 | 验证方式 |
|-------|---------|
| 资金流水 | 本章收支与前文一致 |
| 物品来源 | 本章使用的物品在前文已出现 |
| 交通方式 | 本章交通与前文一致 |
| 人物状态 | 位置/状态与前文衔接 |
| 时间线 | 与日历/季节设定一致 |
