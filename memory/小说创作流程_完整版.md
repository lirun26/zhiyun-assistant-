## ③ 创作阶段

### 🎨 写作五原则（必须遵守）

| 原则 | 说明 |
|:---|:---|
| **声音** | 有立场的叙事者，不是摄像头 |
| **层次** | 情绪有起伏，不是单一色调 |
| **细节** | 动作具体化，有画面感 |
| **呼吸** | 长短句交替，张弛有度 |
| **活性** | 动词有力，少用"的" |

### 🚫 高频错误词（禁止使用）

| 类别 | 禁止词 |
|:---|:---|
| **AI词汇** | 赋能、维度、赛道、迭代、颠覆、众所周知、不言而喻 |
| **虚弱动词** | 进行、进行中、得到、受到、认为、觉得 |
| **连接词** | 然后、之后、接着（用具体动作代替） |
| **感叹词** | 真是太、太好了、真是没想到 |

### 写作规范
- 对话不用"冷笑"、"侥幸"、"淡淡道"等高频词
- 动作描写用具体动词
- 心理活动少用抽象词
- 每段不超过200字
- 章尾留钩子
- **每段第一句不超过15字**

## ④ 评审阶段

### 评审维度

| 角色 | 关注点 | 权重 |
|:---|:---|:---|
| 阅读者 | 开篇吸引力、节奏、画面感 | 25% |
| 编审 | 错别字、病句、一致性 | 25% |
| 故事家 | 剧情逻辑、伏笔、钩子 | 25% |
| 文学顾问 | 语言艺术、人物刻画 | 15% |
| 毒舌读者 | 套路化、水文、毒点 | 10% |

### P0红线（触犯重写）

- ❌ **AI词汇** - "赋能、维度、赛道..."
- ❌ **感悟式结尾** - "经过这次事件，我明白了..."
- ❌ **感叹式结尾** - "真是太...！"
- ❌ **上帝视角** - "所有人没想到..."
- ❌ **时间线矛盾** - 季节/昼夜/日期前后不一致
- ❌ **修为矛盾** - 境界突然变化无合理过程
- ❌ **人物凭空出现** - 前文无铺垫突然出场
- ❌ **状态超前** - 昏迷→清醒、受伤→痊愈、破产→翻盘 → 必须在破局/治愈后

### ⚠️ 逻辑时间线强制检查（每次创作必须）

| 检查项 | 示例 |
|:---|:---|
| **状态变化顺序** | "儿子清醒"必须发生在破局之后，不能在破局前 |
| **时间间隔** | 穿越到商战大佬至少需要数周/数月 |
| **灵力恢复** | 修为恢复必须有过程，不能瞬间满血 |
| **位置连贯** | 不能突然凭空出现在某地，必须有移动过程 |

**错误示例**：
- ❌ 第2章还在昏迷，第3章破局前就"各项指标正常"
- ❌ 三天前觉醒，三天后就被推荐给商业大佬（时间太短）
- ❌ 灵力瞬间恢复

### P1建议避免

- ⚠️ 套路化表达 - "只见一道金光..."
- ⚠️ 流水账叙事 - 事件罗列无起伏
- ⚠️ 抽象心理描写 - "他内心很复杂"
- ⚠️ 重复情节 - 与前文重复
- ⚠️ 超长段落 - 单段落超过300字

### 评分标准

| 分数 | 等级 | 处理 |
|:---|:---|:---|
| 90-100 | 精品 | 可直接发布 |
| 85-89 | 优秀 | 可发布 |
| 75-84 | 良好 | 小改可发布 |
| 60-74 | 合格 | 需修改 |
| 60以下 | 不合格 | 必须重写 |

## ⚠️ 逻辑时间线一致性检查（每次必须执行）

### 检查清单

#### 1️⃣ 时间线检查
- [ ] 上一章的日期/时间是什么？
- [ ] 本章承接的时间点是否衔接？
- [ ] 是否有时间跳跃？跳跃是否合理？
- [ ] 季节、昼夜、节气是否一致？

#### 2️⃣ 角色位置检查
- [ ] 上一章结束时角色在哪里？
- [ ] 本章开头角色是否在同一地点？
- [ ] 角色移动是否需要交代？
- [ ] 移动时间是否合理？

#### 3️⃣ 角色状态检查
- [ ] 上一章结束时角色修为/状态？
- [ ] 本章是否需要状态变化？
- [ ] 状态变化是否有合理过程？
- [ ] 伤势/疲劳是否延续？

#### 4️⃣ 情节逻辑检查
- [ ] 本章事件是否由上一章自然引出？
- [ ] 是否有突兀的剧情转折？
- [ ] 角色决策是否符合其性格？
- [ ] 冲突是否由前文埋下伏笔？

### 常见错误

| 错误类型 | 错误示例 | 正确做法 |
|:---|:---|:---|
| 时间跳跃 | 上一章"三日后"，本章"当天下午" | 明确标注时间变化 |
| 地点突兀 | 上章在宗门，本章突然在皇宫 | 交代如何到达 |
| 修为矛盾 | 上章筑基境，本章炼气境 | 保持修为一致 |
| 人物凭空出现 | 让前文没出场的人物送丹药 | 确认人物在场或引入 |
| 忘记伤势 | 上章重伤，本章活蹦乱跳 | 延续伤势状态 |
| 宝物消失 | 上章获得的宝物不见了 | 保持宝物状态一致 |

## 🤖 AI-Novel-Writing-Assistant 集成

**项目路径**: `/home/admin/.openclaw/workspace/AI-Novel-Writing-Assistant`

### 核心链路（自动导演模式）
```
书级 framing → 故事宏观规划/约束引擎 → 动态角色系统
    ↓
卷战略 → 卷骨架 → 节奏板 → 当前卷拆章
    ↓
章节细化 bundle → 章节执行 runtime
    ↓
state sync → narrative audit/replan → 整本闭环
```

### 启动方式
```bash
cd /home/admin/.openclaw/workspace/AI-Novel-Writing-Assistant
pnpm install    # 首次安装
pnpm dev        # 启动开发服务器
```

### 三种推进模式
1. **按阶段审核** — 每步人工确认
2. **自动推进到可开写** — 生成卷纲后暂停
3. **继续自动执行前10章** — 完全自动

### 对应我们的六步流程
| AI-Novel-Writing-Assistant | 我们的流程 |
|:---|:---|
| 书级 framing + 故事宏观规划 | ①灵感 → ②规划(spec) |
| 动态角色系统 | → 角色状态初始化 |
| 卷战略 + 节奏板 | → 卷纲设计 |
| 章节执行 runtime | ③创作 |
| narrative audit/replan | ④评审 |
| 整本闭环 | ⑥归档 |

### 使用场景
- **新书开局**：用 AI-Novel-Writing-Assistant 的自动导演从灵感开始
- **章节创作**：按我们现有的六步流程严格执行
